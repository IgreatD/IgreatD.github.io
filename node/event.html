# events 事件触发器

大多数Node.js的核心API构建于异步事件驱动架构，当server接收到请求，就把它关闭然后进行处理，然后去服务下一个server。当这个请求完成，它被放回处理队列，当到达队列开头，就把请求的result返回给用户。

这个模型具有高效的性能和扩展性，因为server一直接受请求而不等待任何读写操作。这也称之为非阻塞式IO或者事件驱动IO。

一个简单的EventEmitter实例。

const EventEmitter = require('events');

class MyEmitter extends EventEmitter {}

const myEmitter = new MyEmitter();
myEmitter.on('event', () => {
    console.log('触发事件');
    myEmitter.on('close', () => {
        console.log('事件关闭');
    })
    myEmitter.emit('close')
});
myEmitter.emit('event');
// 触发事件
// 事件关闭

TIP

eventEmitter.emit()用于触发事件，eventEmitter.on()用于接收事件
